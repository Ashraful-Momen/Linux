First install PM2 globally:

```bash id="9o1m7m"
sudo npm install -g pm2
```

Run Odoo with PM2:

```bash id="h3k1s0"
pm2 start "python odoo-bin -c odoo.conf" --name odoo
```

Better approach (recommended):

Use the venv Python directly:

```bash id="oz1f7u"
pm2 start "./venv/bin/python odoo-bin -c odoo.conf" --name odoo
```

Or absolute paths:

```bash id="wyh95n"
pm2 start "/home/ai_root/Project/Smart_Printing_Solution_Ltd/smart_printing_service_limited/venv/bin/python odoo-bin -c odoo.conf" --name odoo
```

Useful PM2 commands:

Check logs:

```bash id="9d6bvf"
pm2 logs odoo
```

Check status:

```bash id="6v2w0v"
pm2 status
```

Restart:

```bash id="m1l3q8"
pm2 restart odoo
```

Stop:

```bash id="m1q5op"
pm2 stop odoo
```

Delete:

```bash id="c8z7rf"
pm2 delete odoo
```

Save PM2 processes after reboot:

```bash id="09g8pd"
pm2 save
```

Enable startup:

```bash id="uj9t2j"
pm2 startup
```

Then run the command PM2 outputs.

Recommended production command for Odoo:

```bash id="m4f9zw"
pm2 start "./venv/bin/python odoo-bin -c odoo.conf" \
--name odoo \
--interpreter none
```

`--interpreter none` avoids PM2 trying to use Node.js interpreter.
